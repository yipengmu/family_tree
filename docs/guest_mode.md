# 游客模式功能说明

## 概述
家谱创作工具现在支持游客模式，用户无需注册或登录即可查看穆家家谱演示。当用户希望创建自己的家谱时，才会引导用户进行注册和登录。

## 功能特性

### 1. 游客模式
- 用户首次访问网站时，默认进入游客模式
- 可以查看默认的穆家家谱数据
- 在家谱页面右上角显示"游客模式"标识
- 提供"创建我的家谱"按钮，引导用户注册

### 2. 数据隔离
- 游客模式下只能查看默认的穆家家谱数据
- 数据库操作（保存、编辑等）需要登录后才能执行
- 租户切换功能在游客模式下受限

### 3. 平滑过渡
- 用户可以在任何时候从游客模式切换到注册/登录
- 创建自己的家谱时，系统会引导完成注册流程
- 之前在游客模式下查看的数据不会丢失

## 技术实现

### 1. 认证状态检测
```javascript
const isAuthenticated = () => {
  const token = localStorage.getItem('token');
  return !!token;
};
```

### 2. 游客模式标识
```javascript
const isGuestMode = () => {
  return !isAuthenticated() && localStorage.getItem('guest_mode') === 'true';
};
```

### 3. 默认租户设置
在游客模式下，系统自动设置为"穆家家谱 (游客模式)"租户。

## 用户体验优化

### 1. 快速加载
- 优先使用缓存数据，快速显示家谱
- 数据库请求设置3秒超时，避免长时间等待
- 后台异步更新数据，不影响用户体验

### 2. 清晰提示
- 在家谱页面右上角显示当前模式状态
- 点击"创建我的家谱"按钮时，引导用户完成注册/登录

### 3. 功能限制
- 需要登录的功能（如设置、创建等）会提示用户先登录
- 提供清晰的登录/注册入口

## 开发注意事项

### 1. 权限控制
- 在关键功能处检查用户认证状态
- 游客模式下调用数据保存等操作会返回错误提示

### 2. 数据管理
- 游客模式下使用默认的穆家家谱数据
- 数据库连接失败时自动回退到本地数据
